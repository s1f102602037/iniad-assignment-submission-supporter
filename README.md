# INIAD課題提出サポーター

INIAD MOOCs の課題ページで提出ボタンを押した日時をブラウザ内に記録し、
一度も押していない場合は画面右下に注意を表示する Chrome / Edge 拡張機能です。

> [!IMPORTANT]
> 本プロジェクトは個人開発の非公式拡張機能です。東洋大学、INIAD、
> INIAD MOOCsの運営者による公式提供・保証はありません。

## 現在の機能

### INIAD MOOCsの提出確認

課題ページ右下のパネルで、提出操作と回答の状態を確認できます。

- 提出ボタンを押した日時を課題ごとに最大20件記録
- 提出操作がない場合は赤色で注意を表示
- 提出後に回答を変更した場合は、再提出が必要であることを警告
- 提出履歴の確認・削除
- 課題の完了状態を手動で設定・解除

### 締め切り管理

提出ボタンがある課題ページで締め切りを登録すると、MOOCsのホーム画面に表示されます。

- 締め切り入力の初期値は、入力日から6日後の23:59
- 講義名・課題名・締め切り・提出状態を締め切り順に表示
- 課題名をクリックして対象ページへ移動
- 締め切りを登録していない課題は一覧の対象外
- 期限終了から24時間が経過した課題は一覧から自動的に非表示

### 外部課題の管理

MOOCs以外の課題もホーム画面右側の一覧で管理できます。

- 課題内容、締め切り日時、任意のリンク先を登録
- 登録済み課題の再編集・削除
- 提出済み・未提出を手動で切り替え
- リンクを登録した課題は、課題名から外部サイトへ移動
- リンク未入力で保存する場合は確認を表示

### 状態の色分け

MOOCs課題と外部課題を共通の色で表示します。

| 色 | 状態 |
| --- | --- |
| 青 | 課題完了・提出済み |
| 黄 | 通常の未完了・未提出 |
| 赤 | 未完了・未提出で締め切りまで24時間以内 |
| 灰 | 提出期限終了 |

### データの保存

- 課題情報はブラウザ内のみに保存
- 開発者や第三者のサーバーへの送信なし

回答変更の判定では、回答そのものではなく比較用の短い指紋を保存します。
以前のバージョンで作成した履歴は残りますが、回答変更の監視は次に提出ボタンを
押した時点から有効になります。

締め切り一覧には、課題ページで日時を登録したものだけが表示されます。
提出ボタンが存在していても、締め切りを登録していない課題や受講していない科目は
一覧の対象になりません。

## GitHubからの導入

1. GitHubのリポジトリ画面で「Code」から「Download ZIP」を選択します。
2. ダウンロードしたZIPファイルを展開します。
3. Chromeで `chrome://extensions/` を開きます。
4. 右上の「デベロッパー モード」を有効にします。
5. 「パッケージ化されていない拡張機能を読み込む」を押します。
6. `manifest.json` が入っている展開済みフォルダを選択します。
7. INIAD MOOCsのページを再読み込みします。

Edgeの場合は `edge://extensions/` から同じ手順で導入できます。

GitHub版は自動更新されません。更新時は新しいZIPを展開し、拡張機能管理画面から
読み込み直してください。

## 権限とプライバシー

本拡張機能が要求する権限は、ブラウザ内へのデータ保存に使う `storage` のみです。
機能は `https://moocs.iniad.org/*` 上でだけ実行されます。

課題履歴や外部課題はブラウザ内の `chrome.storage.local` に保存され、開発者や
第三者のサーバーには送信されません。詳細は [PRIVACY.md](PRIVACY.md) を参照してください。

## 注意事項

この拡張機能が記録するのは「ボタンを押した操作」です。通信エラーや入力エラーなどで
提出が受理されなかった場合も記録される可能性があります。実際の提出結果は必ず
INIAD MOOCs の画面でも確認してください。

INIAD MOOCs 側のボタン表記やURLが変わった場合は、`manifest.json` の対象URLや
`content.js` の `SUBMIT_TEXT_PATTERN` を調整する必要があります。

## 開発

外部ライブラリやビルド処理は使用していません。次の4ファイルが拡張機能本体です。

- `manifest.json`
- `content.js`
- `content.css`
- `README.md`

JavaScriptの構文は次のコマンドで確認できます。

```powershell
node --check content.js
```
